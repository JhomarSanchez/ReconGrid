📋 Formato de Datos de Vehículos:

Para que un archivo sea considerado válido para catalogación, debe contener una columna con datos de vehículos en el siguiente formato:

MARCA MODELO (AÑO_INICIAL-AÑO_FINAL) [ESPECIFICACIONES]

Las especificaciones adicionales (cilindrada, tracción, etc.) son opcionales y se agregarán automáticamente al nombre del vehículo.

✅ Ejemplos de Formatos Válidos:
MAZDA 121 (1996-2001); FORD FIESTA (1996-2001) NISSAN FRONTIER D22 3.0 (1998-2004) FORD-ECOSPORT (2003,2012) 4X2 ← Con tracción después TOYOTA HILUX (2016-) CHEVROLET-N300 (2012,2017) 1.2 ← Con cilindrada después RENAULT-9(1981,1986 ← Sin cierre de paréntesis CHEVROLET-AVEO NG(23) ← Con años de 2 dígitos HYUNDAI-ACCENT (2010,2015) 1.4 SEDAN ← Con múltiples especificaciones
❌ Ejemplos de Formatos NO Válidos:
0 ← Valor numérico sin formato CHEVROLET-LUV 1600 4X2 (-1990) ← Falta año inicial Sin formato definido ← Texto sin estructura MODELO SIN MARCA (2010-2020) ← Debe tener marca
📝 Notas Importantes:
  • El programa busca automáticamente columnas llamadas "VEHICULO MASTER" o "CAR CATALOGUE"
  • Si no se encuentra ninguna de estas columnas, puedes seleccionar manualmente la columna correcta
  • La columna seleccionada será validada analizando si sus datos siguen el formato esperado (mínimo 50% válidos)
  • Múltiples vehículos en una misma celda se separan con punto y coma (;)
  • Soporta años de 2 dígitos (95 = 1995, 23 = 2023)
  • Si no hay año final, se usa el año actual por defecto (ej: 2016-)
  • Información adicional después del paréntesis (cilindrada, tracción, etc.) se incluye automáticamente en el nombre del vehículo
🔍 Proceso de Validación:
  1. El programa detecta automáticamente la columna de vehículos (si existe)
  2. Analiza una muestra de filas para verificar que los datos sean parseables
  3. Si los datos siguen el formato correcto, el archivo es válido
  4. Si no se detecta la columna automáticamente, podrás seleccionarla manualmente
