Formato de columna objetivo:

Para que un archivo sea considerado valido para normalizacion, debe contener una columna con registros semiestructurados similares a este patron:

GRUPO DETALLE (INICIO-FIN) [ATRIBUTOS OPCIONALES]

Los atributos adicionales despues del rango son opcionales y se conservaran como parte del valor normalizado.

Ejemplos de formatos validos:
LINEA ALFA (2019-2021); LINEA BETA (2022-)
SERIE-UNO (2018,2020) DETALLE EXTRA
FAMILIA DOS(23) NOTA OPERATIVA
Ejemplos de formatos no validos:
0
TEXTO SIN ESTRUCTURA DEFINIDA
DETALLE SUELTO (-2020)
Notas importantes:
  • El sistema intenta detectar automaticamente columnas compatibles cuando encuentra encabezados conocidos.
  • Si no detecta una columna util, puedes seleccionar manualmente la columna correcta.
  • La columna seleccionada se valida con una muestra y requiere al menos 50% de filas interpretables.
  • Multiples registros dentro de una misma celda se separan con punto y coma (;).
  • Soporta anos de 2 digitos (95 = 1995, 23 = 2023).
  • Si no hay ano final, se usa el ano actual por defecto.
  • La informacion adicional despues del parentesis se conserva en el valor normalizado.
Proceso de validacion:
  1. El sistema detecta una columna compatible cuando es posible.
  2. Analiza una muestra de filas para verificar que los datos sean interpretables.
  3. Si el patron es consistente, el archivo queda listo para procesar.
  4. Si no se detecta automaticamente, puedes elegir la columna manualmente.
